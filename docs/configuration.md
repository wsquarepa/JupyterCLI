# Configuration

Configuration lives in `config.toml` inside JupyterCLI's config directory. The default location is `~/.config/jhc/config.toml` (`dirs::config_dir()/jhc/`). Set `JHC_CONFIG_DIR` to override the directory.

The file is written with owner-only permissions (`0600`). If it is readable by other users, `jhc` prints a warning and suggests `chmod 600`.

Unknown keys are hard errors: the config is parsed with strict field checking, so a typo fails loudly instead of being ignored.

Most of the time you do not edit this file by hand; `jhc init` and `jhc preset import` write it for you.

## Example Config

```toml
default_hub = "primary"

[hubs.primary]
url = "https://jupyter.example.edu"
token = "abc123..."

[hubs.primary.presets.a100]
profile = "environments"
image = "vscode"
gpus = 2
debug = true

[hubs.backup]
url = "https://other.example.edu"
token = "def456..."
terminal_limit = 2000
```

## Hub Profiles

A config has one or more named hub profiles under `[hubs.<name>]` and a `default_hub` that names which one to use when `--hub` is not given.

| Field | Description |
|---|---|
| `default_hub` | Name of the profile used when `--hub` is not passed |
| `hubs.<name>.url` | Hub base URL, e.g. `https://jupyter.example.edu` |
| `hubs.<name>.token` | API token for the hub |
| `hubs.<name>.terminal_limit` | Optional cap on terminals shown per server; default `999` |
| `hubs.<name>.presets` | Named spawn presets (see below) |

Add a profile with `jhc init --name <name>`. Target a non-default profile per-invocation with `jhc --hub <name> <command>`.

## Tokens

Create a hub token in the browser at `<hub url>/hub/token`, or manage tokens from the CLI:

```sh
jhc token list
jhc token create --note "laptop"
jhc token revoke <id>
```

The `JUPYTERHUB_API_TOKEN` environment variable overrides the stored token for a run, so you can inject a token from a secret store without writing it to disk:

```sh
JUPYTERHUB_API_TOKEN=abc123... jhc status
```

## Spawn Presets

A preset is a named set of spawn options for `jhc start --preset <name>`. Options are stored under `[hubs.<name>.presets.<preset>]` as plain TOML key/value pairs:

```toml
[hubs.primary.presets.a100]
profile = "environments"
image = "vscode"
gpus = 2
debug = true
```

JupyterHub does not expose the list of available environment and resource options over its API, so JupyterCLI cannot enumerate them. Seed a preset by starting a server once from `<hub url>/hub/spawn` in the browser, then capturing it:

```sh
jhc preset import --as a100
```

### Option value formats

When you pass options inline with `-o`/`--option`, the value type is chosen by the separator:

| Form | Meaning | Example | Sent as |
|---|---|---|---|
| `key=value` | String | `profile=gpu` | `"gpu"` |
| `key:=JSON` | Typed JSON | `gpus:=2` | `2` |
| `key:=JSON` | Typed JSON | `debug:=true` | `true` |
| `key:=JSON` | Typed JSON | `args:=["--fast"]` | `["--fast"]` |

Inline `-o` options override matching keys from `--preset` for that run.

## Terminal Limit

`terminal_limit` caps how many terminals the dashboard displays for a server. It defaults to `999` and can be raised or lowered per hub:

```toml
[hubs.primary]
url = "https://jupyter.example.edu"
token = "abc123..."
terminal_limit = 2000
```

When a server has more terminals than the limit, the dashboard says so and directs you to manage the rest with `jhc shell`.
